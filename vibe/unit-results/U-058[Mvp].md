## 1. 작업 요약

핫스팟(Hotspot) 시각적 디자인을 Magenta 테마와 L자 코너 마커 스타일로 개선하여 CRT 터미널 미학에 부합하는 고품질 게임 UI 상호작용 체계를 구축하였습니다.

---

## 2. 작업 범위

- **Hotspot 컴포넌트 분리**: `SceneCanvas` 내부에 있던 핫스팟 렌더링 로직을 독립된 `Hotspot` 컴포넌트로 분리하여 가독성과 성능 최적화(Memoization) 달성
- **Magenta 테마 적용**: 계획서(Option C)에 따라 마젠타/퍼플 계열을 강조색으로 채택하고 상태별(Hover, Active, Drop, Disabled) 색상 변수 정의
- **L자 코너 마커 구현**: CSS Pseudo-elements를 활용하여 4개 모서리에 SF/터미널 스타일의 L자 브라켓 디자인 적용
- **인터랙션 피드백 강화**: 호버 시 펄스 애니메이션, DnD 드롭 타겟 진입 시 앰버 색상 점멸 효과 등 시각적 피드백 고도화
- **접근성 및 반응형 대응**: 키보드 포커스 링, 모바일 터치 영역(최소 44px) 및 툴팁 위치 자동 조정 로직 구현

## 4. 구현 상세

### 4.1 핵심 설계

**디자인 요소**:
- **Primary Color**: `#e040fb` (Magenta) - 터미널 녹색과 대비되는 강력한 상호작용 강조색
- **Corner Markers**: 12px 크기의 L자형 브라켓을 모서리에 배치하여 "Targeting" 시스템 느낌 강조
- **Adaptive Tooltip**: 툴팁이 콘텐츠를 가리지 않도록 상단 배치(기본) 및 모바일 하단 배치 자동 전환

**최적화 및 정합성**:
- **Priority Rendering**: 핫스팟 면적이 작은 것이 항상 위(z-index)에 오도록 정렬하여 겹치는 오브젝트 선택성 개선
- **Performance**: `React.memo`와 CSS `will-change`를 적용하여 다수의 핫스팟 렌더링 시 성능 저하 방지

### 4.2 외부 영향 분석

- **데이터 규약**: (RULE-009) 스타일 변경에도 불구하고 0~1000 정규화 좌표계와 bbox 규약(`[ymin, xmin, ymax, xmax]`)을 엄격히 준수
- **UI 일관성**: 기존 `SceneCanvas` 스타일에서 핫스팟 로직을 완전히 격리하여 유지보수성 향상

## 6. 리스크 및 주의사항

- **색상 대비**: 마젠타 색상이 특정 배경(이미지)에서 대비가 낮아질 수 있으므로, 툴팁에 어두운 반투명 배경을 적용하여 가독성 보강
