## 1. 작업 요약

핫스팟 및 CRT 오버레이의 강도가 콘텐츠를 가리거나 시각적 피로를 유발하는 문제를 해결하기 위해 Option A 전략(면 최소화, 테두리 중심 강조)을 적용하여 팔레트와 강도를 튜닝하고, 모바일 가림 방지를 위한 반응형 툴팁 폴리시를 구현했습니다.

## 2. 작업 범위

- **핫스팟 오버레이 튜닝 (Option A)**: 기본 상태 배경 투명도 대폭 완화(0.05 -> 0.02), 테두리 스타일 변경(dashed), 호버 시 면 강조 대신 테두리/글로우 강화로 콘텐츠 시인성 확보.
- **CRT 효과 최적화**: 전역 스캔라인 밀도(4px -> 2px) 및 플리커 속도(0.15s -> 0.5s) 조정으로 눈의 피로도 감소 및 가독성 레이어링 강화.
- **반응형 툴팁 폴리시**: 모바일(768px 이하)에서 툴팁이 상단 정보를 가리지 않도록 하단 배치로 자동 전환 및 화살표 방향 반전 처리.
- **액션 카드 컴팩트화**: 푸터 높이 조정(160px -> 140px) 및 카드 내부 요소(폰트, 간격) 최적화를 통해 콤팩트한 게임 HUD 인터페이스 강화.
- **접근성 강화**: `prefers-reduced-motion` 대응 시 애니메이션 중단 및 오버레이 강도 추가 완화(0.12) 적용.

## 4. 구현 상세

### 4.1 핵심 설계

**오버레이 강조 전략 (Option A)**:
- **Default**: `background-color: rgba(51, 255, 0, 0.02)`, `border: 1.5px dashed var(--text-color)`
- **Hover**: `background-color: rgba(255, 0, 255, 0.04)`, `border: 2px solid var(--accent-color)`, `transform: none` (콘텐츠 덮임 및 흔들림 방지)

**반응형 툴팁 엔진**:
- 모바일 브레이크포인트(768px)에서 `bottom: auto; top: calc(100% + 6px)`로 강제 전이하여 상단 정보 가림 문제 해결.

**CRT 플리커 밸런스**:
- 0.5s 주기의 완만한 4단계 오퍼시티 변화(`0.95 -> 1.0 -> 0.97 -> 0.99`)를 통해 레트로 자글거림은 유지하되 가독성 저해 요소 제거.

### 4.2 외부 영향 분석

- **UI/UX**: 전체적인 화면 대비가 향상되었으며, 특히 Scene Canvas 내의 이미지를 보면서 핫스팟을 조작하는 경험이 훨씬 선명해짐.
- **성능**: 스캔라인 간격 축소(2px) 및 애니메이션 주기 연장으로 브라우저 렌더링 부담 미세 감소.

### 4.3 가정 및 제약사항

- 핫스팟이 너무 작을 경우(모바일 44px 미만)를 대비하여 CSS에서 `min-width/height` 강제 적용.
- CRT 효과는 `critical` 영역 위에서는 적용되지 않도록 `z-index` 계층 구조 유지.
