# U-057 텍스트 번짐 식별성 개선 실행 가이드

## 1. 개요

CRT 효과로 인한 텍스트 번짐을 개선하여, 터미널 테마 분위기를 유지하면서도 모든 텍스트가 명확히 읽히도록 합니다.

**예상 소요 시간**: 5분

**의존성**:
- 의존 유닛: U-037[Mvp] - CRT/가독성 레이어링 구현
- 선행 완료 필요: 없음 (CSS 변경만 포함)

---

## 2. 빠른 시작 (Quick Start)

### 2.1 환경 준비

```bash
cd frontend
pnpm install
```

### 2.2 즉시 실행

```bash
pnpm dev
```

### 2.3 첫 화면/결과 확인

- 브라우저에서 `http://localhost:8001` 접속
- 프로필 선택 후 게임 화면 진입
- 성공 지표: 모든 텍스트가 번짐 없이 선명하게 표시

---

## 3. 핵심 기능 시나리오

### 시나리오 A: 프로필 선택 화면 텍스트 가독성

**목적**: 프로필 카드 및 안내 텍스트 가독성 검증

**실행**:
1. `http://localhost:8001` 접속
2. 프로필 선택 화면 확인

**확인 포인트**:
- ✅ "UNKNOWN WORLD" 타이틀: 글로우 효과 유지, 가독성 확보
- ✅ "프로필을 선택하세요" 텍스트: 명확히 읽힘
- ✅ 프로필 카드 (서사꾼, 탐험가, 기술 전문가): 제목과 설명 모두 선명
- ✅ 하단 힌트 텍스트: 번짐 없이 표시

---

### 시나리오 B: 게임 UI 텍스트 가독성

**목적**: 주요 게임 UI 영역 텍스트 검증

**실행**:
1. 프로필 선택하여 게임 화면 진입
2. 각 패널별 텍스트 확인

**확인 포인트**:

| 영역 | 확인 항목 | 기대 결과 |
|------|-----------|-----------|
| **Header** | Signal/Shard 수치 | 숫자가 정확히 읽힘 (1, 7, 0 혼동 없음) |
| **Inventory** | 아이템 이름, 수량 | 한글 아이템명 선명, x3 등 수량 표시 명확 |
| **Quest** | 퀘스트 라벨 | 체크박스와 텍스트 구분 명확 |
| **Rule Board** | 규칙 제목, 설명 | 제목은 강조, 설명은 선명 |
| **Agent Console** | Phase 배지, 라벨 | Parse/Validate/Plan 등 즉시 인식 |
| **Narrative** | 타임스탬프, 본문 | [TURN 0] 표시 명확, 본문 읽기 편안 |
| **Action Deck** | 카드 제목, 비용 | 탐색하기/조사하기 선명, Signal 1 등 비용 표시 |

### 시나리오 C: 작은 텍스트 가독성 (Critical)

**목적**: 12px 이하 작은 텍스트 가독성 검증

**실행**:
1. 게임 화면에서 다음 요소들 확인

**확인 포인트**:
- ✅ **비용 표시** (Signal 소모 1): 숫자 명확
- ✅ **위험도 라벨** (위험도 낮음/보통): 텍스트 선명
- ✅ **타임스탬프** ([TURN 0]): 괄호와 숫자 구분 가능
- ✅ **Agent Console 라벨** (대기열, 검증 배지): 작은 라벨도 읽힘
- ✅ **거래 장부 항목**: T1, T2 등 턴 번호와 비용 명확

---

### 시나리오 D: CRT 분위기 유지 확인

**목적**: 가독성 개선이 테마 분위기를 해치지 않았는지 확인

**실행**:
1. 전체 화면 시각적 확인
2. Before/After 비교 (가능한 경우)

**확인 포인트**:
- ✅ **타이틀** (UNKNOWN WORLD): 글로우 효과 유지 (3px 글로우)
- ✅ **패널 헤더** (INVENTORY, QUEST 등): 미세 글로우 유지
- ✅ **Scene Canvas**: CRT 스캔라인 효과 유지
- ✅ **전체 분위기**: "터미널 게임" 느낌 유지

- ✅ CRT 테마 분위기 유지 (과도한 효과 완화, 전체 제거 아님)

- ❌ 너무 밋밋해 보임 → 글로우 값 약간 상향 조정
- ❌ 특정 브라우저에서 다르게 보임 → font-smoothing 속성 확인

## 5. 문제 해결 (Troubleshooting)

### 5.1 일반적인 오류

**오류**: 텍스트가 여전히 번져 보임

- **원인**: text-shadow 값이 여전히 높음
- **해결**: `style.css`에서 해당 요소의 text-shadow를 `none` 또는 더 낮은 값으로 조정

**오류**: 글로우 효과가 너무 약함

- **원인**: text-shadow 값을 너무 낮게 설정
- **해결**: 타이틀/강조 요소에 `0 0 3px rgba(51, 255, 0, 0.4)` 정도의 글로우 복원

### 5.2 환경별 주의사항

- **Windows**: Chrome에서 테스트 권장 (font-smoothing 지원)
- **macOS**: Safari에서 약간 다르게 보일 수 있음 (retina 디스플레이 고려)
- **고해상도**: 더 선명하게 보이므로 문제 없음
- **저해상도**: 글로우 효과가 더 두드러질 수 있음

## 6. 기술 상세

### 6.1 가독성 보호 클래스 (추가됨)

```css
/* 가독성 우선 영역 - CRT 효과 완전 제거 */
.readable-text {
  text-shadow: none !important;
  filter: none !important;
}

/* 분위기 유지 + 가독성 - 미세 글로우 */
.readable-glow {
  text-shadow: 0 0 0.5px currentColor !important;
}

/* 중요 텍스트 강조 - 약간의 글로우 */
.readable-emphasis {
  text-shadow: 0 0 2px rgba(51, 255, 0, 0.3) !important;
}

/* 작은 텍스트 보정 (12px 이하) */
.readable-small {
  font-weight: 500;
  letter-spacing: 0.02em;
}
```

### 6.2 주요 변경 사항

| 요소 | Before | After |
|------|--------|-------|
| body text-shadow | 0 0 5px | 0 0 2px |
| action-card-title | 10px/20px 글로우 | 3px 글로우 |
| phase labels | 5px 글로우 | 2px 글로우 |
| 작은 텍스트 (비용, 배지) | font-weight: 400 | font-weight: 500 |
| phase-item filter | drop-shadow 1px | none |
