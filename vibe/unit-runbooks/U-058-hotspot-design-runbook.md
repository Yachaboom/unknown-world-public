# U-058[Mvp] 핫스팟 디자인 개선 실행 가이드

## 1. 개요

핫스팟(Hotspot) 시각적 디자인을 **Magenta/Purple 테마 + L자 코너 마커** 스타일로 개선하여, CRT 터미널 테마와 조화로우면서도 게임 UI 수준의 시각적 품질을 제공합니다.

**완료 기준**:
- 핫스팟이 Magenta 계열 + L자 코너 마커 스타일로 표시됨
- 호버/포커스/드롭 타겟 상태에서 시각적 피드백 제공
- 기존 bbox 좌표 규약(0~1000) 준수 (RULE-009)

**예상 소요 시간**: 5~10분

**의존성**:
- 의존 유닛: U-010[Mvp] - Scene Canvas 핫스팟 오버레이 기본 구현
- 선행 완료 필요: 프론트엔드 개발 서버 실행 가능 상태

---

## 2. 빠른 시작 (Quick Start)

### 2.1 환경 준비

```bash
# 프론트엔드 의존성 설치
cd frontend
pnpm install
```

### 2.2 개발 서버 실행

```bash
# 프론트엔드 개발 서버 (포트 8001)
pnpm dev
```

- 브라우저에서 `http://localhost:8001` 접속

- Scene Canvas에 핫스팟이 표시되면 새로운 디자인 확인

## 3. 핵심 기능 시나리오

### 시나리오 A: 기본 핫스팟 디자인 확인

**목적**: 새로운 Magenta 테마 + L자 코너 마커 스타일 확인

**실행**:
1. 게임 시작 후 Scene Canvas 영역 확인
2. 핫스팟이 있는 장면으로 이동 (예: 탐색 행동 선택)

**기대 결과**:
- 핫스팟이 **Magenta/Purple 계열 색상**으로 표시됨
- 4개 코너에 **L자 형태 브라켓 마커**가 표시됨
- 반투명 배경 + 얇은 테두리로 영역 표시
- 부드러운 마젠타 글로우 효과

**확인 포인트**:
- ✅ 핫스팟 테두리 색상이 `#e040fb` (밝은 마젠타)
- ✅ 코너 마커가 L자 형태로 4개 모서리에 표시
- ✅ 기존 녹색 스타일이 아닌 새로운 마젠타 스타일

---

### 시나리오 B: 호버 상태 피드백 확인

**목적**: 마우스 호버 시 시각적 피드백 검증

**실행**:
1. 핫스팟 위로 마우스 커서 이동
2. 시각적 변화 관찰

**기대 결과**:
- 핫스팟 색상이 `#f06292` (핑크)로 밝아짐
- 글로우 효과 강화 (더 넓은 범위)
- 미세한 스케일 확대 (scale 1.01)
- 코너 마커에 미세한 펄스 애니메이션

**확인 포인트**:
- ✅ 호버 시 색상 변화가 명확함
- ✅ 툴팁이 마젠타 테두리로 표시됨
- ✅ 부드러운 전환 애니메이션 (0.2s)

---

### 시나리오 C: 드롭 타겟 상태 확인 (DnD)

**목적**: 인벤토리 아이템 드래그 시 드롭 타겟 시각화 검증

**실행**:
1. 인벤토리에 아이템이 있는 상태에서 아이템 드래그 시작
2. 핫스팟 위로 드래그

**기대 결과**:
- 핫스팟 색상이 `#ffab40` (앰버/골드)로 변경
- 강화된 글로우 + 점멸 애니메이션
- 코너 마커도 앰버 색상으로 변경
- 툴팁에 "DROP HERE" 힌트 표시

**확인 포인트**:
- ✅ 드롭 타겟 상태가 호버와 명확히 구분됨
- ✅ 앰버 색상의 점멸 효과 (0.8s 주기)
- ✅ 드롭 힌트 텍스트가 표시됨

---

### 시나리오 D: 비활성화 상태 확인

**목적**: 스트리밍 중 핫스팟 비활성화 시각화 검증

**실행**:
1. 행동 카드 선택 후 스트리밍 시작
2. 스트리밍 중 핫스팟 상태 확인

**기대 결과**:
- 핫스팟 색상이 `#9c27b0` (어두운 퍼플)로 흐려짐
- 테두리가 점선으로 변경
- 반투명도 50%
- 호버해도 반응 없음

**확인 포인트**:
- ✅ 비활성화 상태가 시각적으로 명확함
- ✅ 커서가 `not-allowed`로 변경
- ✅ 클릭 불가

---

### 시나리오 E: 접근성 확인 (키보드/reduced-motion)

**목적**: 키보드 접근성 및 reduced-motion 지원 확인

**실행**:
1. Tab 키로 핫스팟에 포커스 이동
2. Enter 또는 Space로 클릭
3. (선택) 브라우저/OS의 reduced-motion 설정 활성화

**기대 결과**:
- 포커스 시 `outline` 표시 (2px 마젠타)
- Enter/Space로 정상 클릭 동작
- reduced-motion 시 애니메이션 비활성화

**확인 포인트**:
- ✅ 키보드만으로 핫스팟 상호작용 가능
- ✅ 포커스 링이 명확하게 표시됨
- ✅ reduced-motion 환경에서 과도한 애니메이션 없음

## 5. 문제 해결 (Troubleshooting)

### 5.1 일반적인 오류

**오류**: 핫스팟 스타일이 적용되지 않음
- **원인**: CSS 파일 import 누락
- **해결**: `Hotspot.tsx`에서 `import '../styles/hotspot.css'` 확인

**오류**: L자 코너 마커가 보이지 않음
- **원인**: 핫스팟 크기가 너무 작음 (코너 마커 크기 12px)
- **해결**: 핫스팟 영역이 충분히 큰지 확인 (최소 40x40px 권장)

**오류**: 애니메이션이 버벅임
- **원인**: 다수의 핫스팟 동시 렌더링
- **해결**: `will-change: transform, box-shadow` 적용 확인 (GPU 가속)

### 5.2 환경별 주의사항

- **Windows**: 정상 동작
- **macOS/Linux**: 정상 동작
- **모바일**: 터치 영역 최소 44px 보장됨
